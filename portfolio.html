Descripción General

¿Qué es Lumière Store?

Una tienda de moda en línea completa construida sin framework de backend propio. Toda la infraestructura está gestionada por servicios cloud modernos, permitiendo escalar sin mantenimiento de servidores.

🛍️

Catálogo Dinámico

Productos cargados desde Supabase con filtros por categoría, estado de oferta y búsqueda en tiempo real.

🔐

Autenticación Completa

Registro, inicio de sesión y sesiones persistentes usando Supabase Auth. El carrito de invitado se fusiona al loguearse.

💳

Pagos con Stripe

Checkout seguro alojado en Stripe. Las órdenes se confirman automáticamente via webhook y se guardan en la BD.

🛒

Carrito Inteligente

Persiste en localStorage para invitados y en Supabase para usuarios. Soporte de cantidades, tallas y colores.

📦

Historial de Órdenes

Los usuarios autenticados pueden ver sus pedidos anteriores con estado, productos y montos detallados.

📱

100% Responsive

Diseño adaptable a móviles, tablets y escritorio. Menú hamburguesa, grid fluido y tipografía escalable.

// Edge Function — crea sesión de pago en Stripe
import Stripe from 'https://esm.sh/stripe@14?target=deno';

const stripe = new Stripe(Deno.env.get('STRIPE_SECRET_KEY'));

serve(async (req) => {
  const { items, success_url, cancel_url } = await req.json();

  const session = await stripe.checkout.sessions.create({
    payment_method_types: ['card'],
    line_items: items.map(i => ({ /* ... */ })),
    mode: 'payment',
    success_url,
    cancel_url,
  });

  return new Response(JSON.stringify({ url: session.url }));
});

Cómo Funciona

Flujo completo desde que el usuario visita la tienda hasta que se confirma el pago.

🌐  Navegador del Usuario
HTML · CSS · JavaScript
⬇ SDK · REST ⬇
🟢  Supabase Auth
JWT · Sesiones
🗄️  PostgreSQL + RLS
Productos · Carrito · Órdenes

create-checkout · stripe-webhook
⬇ API Segura ⬇
💜  Stripe Checkout + Webhooks
Pago · Confirmación · Reembolsos
1️⃣
Usuario navega
Productos cargados desde Supabase. Invitados usan localStorage.
2️⃣
Se autentica
Supabase Auth emite JWT. El carrito migra de local a base de datos.
3️⃣
Checkout
Edge Function crea sesión en Stripe. Usuario paga de forma segura.
4️⃣
Confirmación
Webhook de Stripe guarda la orden en PostgreSQL y vacía el carrito.

Funcionalidades

Qué puede hacer el usuario

🛍️

Explorar productos

Catálogo con 8 productos reales, filtros por categoría (Mujer, Hombre, Accesorios, Sale) y vista rápida sin recargar la página.

🔍

Búsqueda rápida

Barra de búsqueda deslizable en el header. Soporta teclado (Enter para buscar, Escape para cerrar).

🛒

Carrito persistente

Añadir, eliminar y cambiar cantidades. Se guarda en localStorage (invitado) o Supabase (usuario autenticado).

👤

Cuenta de usuario

Registro con nombre y email, inicio de sesión, cierre de sesión. El carrito de invitado se fusiona automáticamente al entrar.

💳

Pago con Stripe

Checkout seguro alojado en Stripe. Acepta tarjetas de crédito/débito. Confirmación automática por webhook.

📦

Historial de órdenes

Panel de órdenes con estado visual (pendiente, pagado, enviado, entregado) y desglose de productos comprados.

♥

Lista de deseos

Agregar y quitar productos de favoritos con feedback visual inmediato mediante toast notifications.

📧

Newsletter

Formulario de suscripción con validación de email. Preparado para conectarse con servicios de email marketing.

Proceso de Desarrollo

Fases del Proyecto

Fase 1
Diseño UI / Maquetación
Diseño de la interfaz completa: header sticky, hero slider, grid de colecciones, carrito deslizante, modales y footer. Inspirado en tiendas Shopify premium.
HTML5 CSS3 Responsive Google Fonts
Fase 2
Lógica Frontend Interactiva
Carrito funcional, filtros de productos, vista rápida modal, slider automático, búsqueda, scroll suave y animaciones con IntersectionObserver.
JavaScript ES2022 localStorage IntersectionObserver
Fase 3
Base de Datos y Autenticación
Diseño del schema PostgreSQL con políticas RLS, tablas para productos, carrito, órdenes y perfiles. Trigger automático para crear perfil al registrarse.
Supabase PostgreSQL RLS Auth JWT
Fase 4
Pagos y Edge Functions
Integración de Stripe Checkout mediante Edge Functions en Deno/TypeScript. Webhook para confirmar pagos y guardar órdenes automáticamente.
Stripe API Deno TypeScript Webhooks
Fase 5
Integración y Corrección de Errores
Unificación del frontend con el backend. Corrección de errores de carga de módulos ES, manejo de rutas file://, fallback a datos estáticos y guard isConfigured().
Debugging CDN UMD Error Handling Fallback
